# 激进版 MACD 策略说明

本文档只描述当前仓库正在运行的这一版策略。

## 策略目标

这套策略不是追求平滑曲线，而是追求在明显趋势里更早上车、尽量陪跑、在趋势失效时尽快离场。

它接受更高的波动和更深的回撤，但会尽量避开三类坏场景：

- 横盘来回扫
- 没有跟随质量的假突破 / 假破位
- 走了一点就衰竭的弱延续

## 策略使用哪些数据

当前主数据全部来自 `OKX`：

- `15m`
  唯一事实层，也是策略真正决策的主时间框架
- `1h / 4h`
  由 `15m` 聚合得到，只做趋势方向、环境和 flow 确认
- `1m`
  只给回测器做更细的执行价近似
- funding
  只在回测结算里计入

策略和回测器当前会实际使用这些量：

- 价格 `open / high / low / close`
- 总成交量 `volume`
- 计价成交额 `quote_volume`
- 方向流量代理 `trade_count / taker_buy_volume / taker_sell_volume`
- 由上面这些量派生出来的 `taker_buy_ratio / taker_sell_ratio / flow_imbalance`

说明：

- `trade_count` 和 taker 方向量在 OKX 公共 K 线里没有原始字段，当前是用 K 线形态和成交额推导的稳定代理值。
- 当前 `train / val` funding 覆盖率仍是 `0%`，缺失区间按 `0 funding` 处理。

## 决策链

策略现在是一条有名字的规则链，不再是超长条件直接堆在 `strategy()` 里。

### 1. 先过滤横盘

先看当前是不是“不值得做”的市场：

- 波动太小
- EMA 扩散太弱
- ADX 太低
- chop 太高

只要环境太像横盘，就直接不做。

### 2. 再看外层方向环境

横盘过滤通过后，长侧和短侧各自会先看外层环境是否允许继续：

- `long_outer_context_ok`
- `short_outer_context_ok`

这里主要看：

- `15m / 1h / 4h` 的方向是否大体一致
- 趋势质量是否过线
- flow 是否没有明显逆风

### 3. 再看具体入场路径

长侧当前会检查：

- `long_breakout_ok`
- `long_pullback_ok`
- `long_trend_reaccel_ok`
- `long_ownership_relay`

短侧当前会检查：

- `short_breakdown_ok`
- `short_bounce_fail_ok`
- `short_trend_reaccel_ok`

这些路径不是分别对应多套独立策略，而是同一套趋势思路下的几种进场方式。

### 4. 最后过一层 veto

即使方向和路径都通过，策略还会再做最后一层否决：

- `long_final_veto_clear`
- `short_final_veto_clear`

这层主要挡掉已经太晚、质量不足、延续风险过高、或者 flow 明显转坏的情况。

## 当前对外信号

从 `strategy()` 的真实返回值看，当前对外只有两类执行主标签：

- `long_pullback`
- `short_breakdown`

这不代表长侧只会做“回踩”，而是说明长侧多条路径最终会汇到同一个执行标签上。

为了让研究器和实盘壳能看清到底是哪条路径在起作用，当前同时会输出路径标签：

- 多头：`long_impulse / long_retest / long_reaccel / long_relay`
- 空头：`short_impulse / short_retest / short_reaccel`

主标签负责执行语义和退出参数族；路径标签只负责诊断、统计和 `freqtrade enter_tag`。

## 风控与退出

当前执行和风控主线包括：

- `20x` 杠杆
- 单仓保证金占比 `17%`
- 单仓最小保证金 `5000`
- 单仓最大保证金 `30000`
- 最大并发仓位 `4`
- 最大加仓次数 `2`

说明：当前退出参数位于策略脚本的 `EXIT_PARAMS`。研究器允许 AI 调整止盈、止损、保本、追踪、持仓时间、趋势失效退出和加仓触发；不允许 AI 调整杠杆、仓位比例、单仓上下限、最大并发仓位和加仓规模。

退出链包括：

- ATR 初始止损
- `TP1` 分批止盈
- 保本止损
- 追踪止损
- 趋势失效退出
- 时间退出
- 反向信号退出

## 回测口径

当前回测器会计入：

- `1m` 执行价近似
- 滑点
- 手续费
- funding
- 多并发仓位
- 有限次加仓

交易统计口径按“整笔仓位”算，不会把 `TP1` 拆成独立 trade。

## 当前策略画像

从当前 champion 指标看，这版策略的空头捕获明显强于多头捕获。

简单说就是：

- 空头方向已经能比较稳定地抓到一部分下跌
- 多头方向还不够强，经常出现没跟上、陪跑不够，或者过早退出

因此当前研究器更值得优先看的位置，通常会集中在：

- 长侧外层环境是否太苛刻
- 长侧最终 veto 是否太早挡掉
- 长侧延续确认是否不够敢跟
